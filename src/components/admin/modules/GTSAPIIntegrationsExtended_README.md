# GTS API Integrations Extended Module

## Описание
Комплексный модуль для управления внешними интеграциями, вебхуками и автоматизацией процессов в GTS Executive Panel. Предоставляет разработчикам и администраторам единый интерфейс для подключения к внешним системам и мониторинга их работы.

## Основные функции

### 1. Каталог интеграций
**Доступные интеграции:**
- **CRM Integration**: YClients или другие CRM системы для обмена лидами и бронированиями
- **Marketing Channels**: 
  - Avito (автоматическое размещение объявлений)
  - Instagram Business (публикация контента и статистика)
  - VK, YouTube, TikTok, Яндекс.Дзен (планируется)
  - Telegram Bot (уведомления и поддержка)
- **Analytics**: Google Analytics, Яндекс.Метрика для импорта данных
- **Payment Providers**: В разработке
- **Telephony**: IP-телефония (в разработке)

**Функции:**
- Статус подключения (подключено/отключено/ошибка/в разработке)
- Метрики запросов (общее количество, успешные, ошибки)
- Последняя синхронизация
- Тестирование подключения
- Настройка параметров

### 2. Управление вебхуками
**События:**
- Новая бронь (`booking.created`)
- Изменение статуса брони (`booking.updated`)
- Новый лид из маркетинга (`lead.created`)
- Обновление техники (`fleet.updated`)
- Успешный импорт данных из CRM (`crm.import.completed`)

**Функциональность:**
- Настройка URL получателя и метода (POST/PUT/PATCH)
- Включение/отключение вебхуков
- Выбор формата данных (JSON/XML/Form)
- Статистика успешных/неудачных отправок
- Мониторинг времени ответа
- Тестирование вебхуков
- Фильтрация по статусу и типу события

### 3. Автоматизации процессов
**Доступные сценарии:**
- **Триггер**: Новый лид в CRM → **Действие**: Отправить уведомление в Slack
- **Триггер**: Изменение доступности техники → **Действие**: Обновить Avito объявления
- **Триггер**: Расписание (ежедневно 09:00) → **Действие**: Импорт из YClients
- **Триггер**: Расписание (каждый час) → **Действие**: Обновление статистики

**Возможности:**
- Конструктор сценариев "если-то"
- Настройка расписания для периодических задач
- Мониторинг выполнения и ошибок
- Включение/отключение автоматизаций

### 4. Мониторинг API
**Метрики:**
- Общее количество запросов
- Количество успешных/неудачных запросов  
- Среднее время ответа
- Статус всех интеграций в реальном времени

**Логирование:**
- История всех запросов и ответов
- Фильтрация по интеграции и коду ответа
- Детали ошибок с временными метками
- Экспорт логов

**Уведомления:**
- Превышение лимитов API
- Недоступность внешних сервисов
- Критические ошибки интеграций

### 5. Управление ключами и средами
**API ключи:**
- Генерация внутренних API ключей для мобильных приложений и партнёров
- Настройка прав доступа (чтение/запись по модулям)
- Отслеживание использования ключей
- Установка сроков действия

**Среды:**
- Переключение между Test и Production
- Раздельные настройки для разработки и боевого окружения
- Изоляция тестовых данных

**Права доступа:**
- Executive: Полный доступ ко всем функциям
- Разработчики/DevOps: Создание интеграций, управление ключами
- IT-администраторы: Настройка и мониторинг
- Менеджеры маркетинга/CRM: Просмотр статуса без изменений

## Технические особенности

### Архитектура
- Модульная система интеграций
- Асинхронная обработка вебхуков
- Централизованное логирование
- Система повторных попыток для failed запросов

### Безопасность
- Шифрование API ключей
- Валидация webhook подписей
- Rate limiting для предотвращения злоупотреблений
- Аудит всех действий

### Мониторинг и алерты
- Real-time мониторинг статуса интеграций
- Автоматические уведомления при сбоях
- Метрики производительности
- SLA мониторинг

## Интеграция с системой

Модуль полностью интегрирован в GTSExecutivePanel:
- Вкладка "Integrations" с иконкой Zap
- Роль-ориентированный доступ
- Единая дизайн-система GTS
- Мок-данные для демонстрации функций
